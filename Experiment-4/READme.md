
# Experiment 4 — Speeding Up Data Transfer Using S3 Transfer Acceleration

## Aim

To demonstrate faster data upload to Amazon S3 using the Transfer Acceleration feature.

## Requirements

- AWS Account
- Amazon S3 Bucket
- Internet Browser
- Sample file for upload

## Services Used

- Amazon S3
- S3 Transfer Acceleration

## Theory

Amazon S3 Transfer Acceleration improves file upload speed by routing data through the nearest AWS Edge Location and then through the AWS global network infrastructure.

Normal Transfer:
Client → S3 Bucket

Accelerated Transfer:
Client → AWS Edge Location → AWS Global Network → S3 Bucket

Advantages:
- Reduced latency
- Faster long-distance uploads
- Improved large file transfer performance

## Procedure

1. Login to AWS Console.

2. Open Amazon S3 service.

3. Create a new S3 bucket.

4. Configure bucket settings:
   - Unique bucket name
   - Default region
   - Default bucket configuration

5. Click Create Bucket.

6. Open the created bucket.

7. Go to:
   - Properties
   - Transfer Acceleration

8. Click Edit.

9. Enable:
   - Transfer Acceleration

10. Save changes.

11. Upload files using normal S3 upload method.

12. Observe the upload process.

13. Use the accelerated endpoint provided by AWS:
    - bucketname.s3-accelerate.amazonaws.com

14. Upload files using Transfer Acceleration.

15. Observe upload performance.

## Observation

Transfer Acceleration helps improve upload speed by routing traffic through AWS edge locations and AWS global network infrastructure.

## Result

Successfully enabled and demonstrated S3 Transfer Acceleration using Amazon S3.
